## 결정 상태 정의
- ✅ **채택** — 실제 프로젝트에 적용
- ❌ **기각** — 현재 단계에서 사용 안 함 (이유 명시)
- ⏸️ **보류** — 추후 재검토 (시기 명시)
- 🔄 **검토중** — 아직 결정 안 됨

## 결정 템플릿

새 결정을 추가할 때 아래 형식으로 `decisions/YYYY-MM-DD_<주제>.md` 파일을 만드세요.

```markdown
# [결정] <주제>

- **날짜**: YYYY-MM-DD
- **결정**: ✅ 채택 / ❌ 기각 / ⏸️ 보류
- **결정자**: 이름
- **적용 시기**: Phase X

## 배경
왜 이 기술/방법을 검토하게 됐는지

## 검토 내용
- 장점:
- 단점:
- 비용/복잡도:

## 결정 근거
최종적으로 이렇게 결정한 이유

## 다음 액션
- [ ] 담당자: 할 일
```
